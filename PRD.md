## 1. 一句话定位

以《西湖繁胜全景图》长卷为核心，制作一个兼具“题名入境、鸟瞰入画、长卷漫游、史料考证、社区共研”的沉浸式交互网站；首版 MVP 以“西湖北岸”部分画卷为范围，先验证从 Landing Page 进入南宋西湖全局示意地图，再进入长卷、横向游湖、查看节点、阅读考证和参与问湖共创的体验闭环。

## 2. 产品背景

《西湖繁胜全景图》作为长卷图像，天然具有连续空间、游观节奏和叙事路径。普通图片查看方式会削弱长卷的尺度感、行旅感和细节发现过程。本项目要把“看一张图”升级为“进入一段可漫游的历史空间”：用户先经由独立 Landing Page 完成题名入境和气质预热，再在 `入画` 中俯瞰一个由长卷推演出的南宋西湖全局示意地图，从鸟瞰图上可见的重要入口点进入对应游湖区域，在 `游湖` 中横向漫游，点击更细的节点理解图像细节，并通过 `拾遗` 与 `问湖` 建立更完整、可持续生长的知识层。

当前已有 MVP 原始素材包：`data/raw/西湖北岸/`，其中包含西湖北岸大图、全景鸟瞰标注图、切片文件、北岸长卷文字标注参考图和局部动图。首版不追求完整 16 米长卷一次做完，而是先用当前已确定的静态网页初稿验证“Landing 入境 → 入画分卷入口 → 西湖北岸三段横卷 → 节点题签与信息卡片 → 拾遗参考图页 → 问湖冷知识图册”的体验闭环。

## 3. 核心问题

- 用户如何先被项目气质和题名吸引，再从全局鸟瞰理解南宋西湖的大致空间，并进入局部长卷游览？
- 长卷图像很长，用户如何在网页中自然地横向游览，而不是被普通滚动页面打断？
- 图像细节很多，用户如何在不破坏沉浸感的前提下获得节点解释？
- 西湖场景有声音想象，但当前初稿先去掉声景入口；后续如恢复，如何让声景不抢占长卷主视觉？
- 画作背景和流传考证需要严谨，如何把“可读性”和“学术可信度”同时建立起来？
- 项目会不断遇到未定问题和新线索，如何让用户参与提问、讨论和提供资料，同时不破坏内容可信度？

## 5. 产品目标

### 5.1 MVP 目标

1. 完成一个从 Landing Page 进入 `入画` 南宋西湖鸟瞰示意地图，再进入 `游湖` 长卷位置的路径闭环。
2. 建立 `巨石山下大石佛` 横卷浏览基础交互模型，包括底部滑块、鼠标拖拽、鼠标滚轮缩放和键盘辅助。
3. 建立节点信息卡片模型；当前 15 个西湖北岸节点已进入可视化题签与卡片体系，后续重点是继续校准坐标、补证据来源和完善文案。
4. 当前初稿不实现声景入口和声景联动；声景从 P0 移至后续确认项。
5. 建立 `拾遗` 的学术内容结构，首版允许部分内容标记为“待考证”，但不得把未核实信息写成定论。
6. 建立 `问湖` 的冷知识图册展示结构，支持分区导航、更多展开、整包下载和大图轮播预览；社区共研结构保留为后续扩展。
7. 当前 MVP 只验收桌面 Web 端体验，优先保证 `1024px` 以上视口下的浏览、点击、拖拽、滚轮缩放、键盘与弹层可用。

### 5.2 长期目标

1. 扩展到完整 16 米长卷，形成可持续追加地图点、长卷节点、史料、音频和社区线索的内容系统。
2. 形成“题名入境 + 鸟瞰导航 + 图像游观 + 学术注释 + 社区共研”的沉浸式数字长卷范式；声景叙事作为可选增强项后置。
3. 支持展陈、课堂、社交传播、公众共创和个人研究整理等多场景复用。

## 6. 非目标

- MVP 不要求完成完整 16 米长卷全量数字化体验。
- MVP 不要求完成严格 GIS 精准复原；`入画` 地图是基于长卷和后续史料逐步校准的艺术化/示意性环湖鸟瞰图。
- MVP 不编造画作年代、作者、收藏、流传等史实；没有来源的内容必须标注“待考证”。
- MVP 不构建复杂账号体系、积分系统或全量社区后台；`问湖` 当前首版以冷知识图册与静态资源展示为主，结构化留言、线索提交与人工整理流程后置。
- MVP 不直接把 141MB 级别原始大图作为线上页面资源；需要生成 Web 可用素材。
- MVP 不做移动端页面适配、触控手势体验、移动端导航重排和移动端视觉验收；这些进入后续响应式专项。

## 7. 信息架构

网站结构为“前置 Landing Page + 四个一级版块”。Landing Page 不计入四个一级版块，它是进入项目的封面、题名和动效入口；进入后再展开 `入画`、`游湖`、`拾遗`、`问湖` 四个一级版块：

0. Landing Page / Entry：前置沉浸封面，当前使用已确认底图，叠加四个版块入口热区。
1. 入画 Bird's-eye Map：进入后的第一个一级版块，呈现基于长卷生成的南宋西湖全局鸟瞰示意地图。用户点击鸟瞰图上的重要入口点，从该点进入 `游湖` 长卷对应区域或锚点。
2. 游湖 The Scroll：核心体验区，当前承载 `巨石山下大石佛` 横卷浏览、节点热区和信息卡片；随动声景后置。
3. 拾遗 Historical Notes：学术背景、图像解读、流传考证和来源索引。
4. 问湖 Cold Knowledge / Inquiry：当前首版承载“西湖繁胜冷知识集”图册墙；长期再扩展为关于南宋西湖的问题、讨论、线索提交和资料共建。

推荐首版导航方式：Landing 使用全屏题名和进入动作；进入主体验后使用顶部或侧边极简文字导航 + `入画` 地图点入口 + `游湖` 卷轴进度。视觉上避免现代 SaaS 式导航栏，采用纸墨底、细线分割、朱印式当前状态提示。

## 8. 版块需求

### 8.1 Landing Page / Entry

目标：在四个一级版块之前建立一个独立的沉浸式入口，让用户先感到“入卷”的气息，再选择进入主体验。当前 Landing 以已确认底图为准，不再按早期 `landing-page-web-effect-v2` 重新排版。

核心需求：

- Landing 是网站第一屏，位于 `入画`、`游湖`、`拾遗`、`问湖` 四个一级版块之前。
- 首屏必须明确出现项目题名“西湖繁胜全景图”，题名是第一视觉信号，不只放在导航或角标中。
- 当前初稿直接使用已确认的整屏底图，不再额外重构树影、云纹、题名、章序等视觉元素。
- 当前初稿只叠加四个版块入口文字热区；`入画`、`游湖`、`拾遗`、`问湖` 都可直接点击进入，不再单独设置“入卷/直接入画”按钮。
- 动效当前保持克制：可有极轻背景漂移或 hover 变朱砂，但不得新增与底图冲突的竖条、玻璃拟态、强渐变或现代装饰。
- Landing 不再额外承载说明长句，避免破坏底图已完成的版式。
- 当前初稿不提供声音入口。

MVP 验收：

- 用户能在 3 秒内知道这是《西湖繁胜全景图》的入口页。
- 用户能清楚点击四个版块入口，默认主路径仍是进入 `入画`。
- Landing 与后续 `入画`、`游湖` 视觉气质连续，不像孤立的营销页。
- 底图和入口热区不影响首屏可交互时间；低性能设备可保持静态封面。
- 桌面 Web 视口下题名、入口和底图不互相遮挡；移动端遮挡问题后置处理。

### 8.2 入画 Bird's-eye Map

目标：让用户先从全局俯瞰进入南宋西湖，再通过地图点进入对应长卷位置。`入画` 不再只是氛围门户，而是整个体验的空间入口。

核心需求：

- `入画` 是 Landing 之后的第一个一级版块，当前主体直接使用 `design/ChatGPT Image 2026年7月1日 21_00_59.png` 作为全屏底图。
- 地图采用示意性环湖地图，不追求 GIS 精度，但要让用户理解西湖环湖结构、北岸范围和可进入点。
- 地图视觉保持宣纸、淡墨、朱印点睛和古籍网格气质，避免现代旅游地图、高饱和底图或商业地图样式。
- 地图点当前实现为透明热区，覆盖底图中红色分卷文字区域。点击热区后跳转到 `游湖` 中对应区域、`scrollX` 或最近节点锚点。
- 地图点以 `data/raw/西湖北岸/标注版本西湖全景鸟瞰.jpg` 中已标注的重要节点为录入依据；`标注版本西湖全景鸟瞰.psd` 作为可编辑原始参考保留，不直接用于 Web。
- MVP 入画地图点不强制与 15 个游湖细节点一一对应；部分游湖节点在鸟瞰图上不可见或不适合在全景层级标出，应只在 `游湖` 长卷中出现。
- 每个入画地图点需要配置 `scrollTarget`，目标可以是游湖区域 id、归一化 `scrollX` 或最近的游湖节点 id。
- 版块内可保留小题签和少量说明，但不再承担 Landing 的大标题海报职责；核心交互是“从地图入画”。
- 地图热区应保留轻量 hover/focus 状态，但不额外覆盖大浮层，避免破坏参考图底图版式。
- MVP 先突出“西湖北岸”可进入区域；全湖其他区域可淡化为未开放/待扩展状态。
- 当前 `入画` 不提供声音开启入口。

推荐地图点信息字段：

```yaml
id: map-north-bank-001
label: 地图点名称
region: 西湖北岸
status: open | locked | pending
scrollTarget: north-bank-region-or-scroll-x-or-node-id
targetType: region | scrollX | node
sourceStatus: 已证实/较可信/待考证/创作性推演
summary: 进入前的 20-40 字提示
```

MVP 验收：

- 用户从 Landing 进入后，能在 3 秒内理解这里是进入长卷的南宋西湖全局地图。
- 用户能点击鸟瞰图上标注的重要入口点进入 `游湖` 的对应区域或锚点。
- 地图以参考图语境表达“导览/分卷入口”，不作为精准 GIS 复原。
- 桌面 Web 热区命中区域稳定，可被鼠标点击和键盘聚焦；移动端触控尺寸后置。
- 该版块不出现现代大卡片、强渐变、玻璃拟态、过度装饰和 emoji。

### 8.3 游湖 The Scroll

目标：让用户像展开长卷一样横向游览西湖北岸，并在游览中发现节点。

核心需求：

- 横向漫游：当前支持底部滑块、鼠标拖拽、滚轮缩放和键盘左右键；滚轮不再作为横向移动主控，而是用于放大/缩小横卷。
- 长卷承载：当前使用 `data/raw/西湖北岸/西湖北岸切片/孤山四圣延祥观.jpg`、`葛岭寺观平章第.jpg`、`巨石山下大石佛.jpg` 三段横卷拼接展示，优先保证画面清晰、移动稳定、不卡顿。
- 有限横卷：当前初稿不实现无缝循环；以三段有限横卷范围、滑块进度和拖拽平移为准。
- 进度提示：底部提供低干扰的横向滑块，用于左右移动横卷。
- 节点热点：画面上可点击节点以朱砂纸签表现，文字竖版单列，优先覆盖底图上的同名文字，不使用现代图钉。
- 信息卡片：点击节点后出现纸墨质感信息层。当前 `cardNode` 节点统一使用大标题、状态签、拼音小标、位置标签、四段正文和朱印的专属卡片样式；长标题需自动压缩或去掉括号别名。
- 关闭与返回：信息卡片必须能通过关闭按钮、Esc、点击遮罩或返回滚动画面恢复沉浸。
- 声景联动：当前初稿移除声景按钮和声景图标，后续如恢复需重新设计入口与音频授权。
- 进入来源：支持从 `入画` 地图点带着 `scrollTarget` 进入指定区域、`scrollX` 或最近节点位置。
- 素材基础：当前直接引用 `data/raw/西湖北岸/西湖北岸切片/` 中的三段北岸切片作为游湖横卷底图；后续再做 Web 优化版本。

#### 8.3.1 节点卡片样例：断桥

`断桥` 卡片是当前节点卡片样式和正文结构的样例之一，后续开新窗口继续时可按此结构扩展其他节点：

```text
【节点简介】
位于白堤东端，跨西湖内湖与外湖，是南宋西湖北岸连通临安城区的核心景观桥梁，也是白堤的标志性起点。

【南宋建制】
南宋时正式称断桥，为石墩木梁式平桥，桥畔建有亭阁，是钱塘门入城游湖的第一节点，属西湖核心地标景观。

【今日坐标】
位于今西湖白堤东端，现存桥梁为后世改建，格局与南宋略有差异，为西湖核心标志性景观。

【史迹价值】
是南宋西湖“一湖三堤”景观格局的核心组成节点，见证了八百年来西湖堤桥体系的沿革，承载着西湖文化的核心记忆。
```

卡片版式要求：

- 标题为大字横排“断桥”，下方小字为 `DUAN · QIAO`。
- 顶部状态签为“南宋 · 西湖堤桥”。
- 位置标签为“南宋西湖北岸 / 白堤东端 / 核心景观桥梁”。
- 正文分四段横排，不强行竖排。
- 右下角保留“桥”朱印。
- 不显示右上角竖版“断桥”。
- 不显示标题区装饰横线。
- 不显示通用卡片的“类别/来源/查看拾遗/问湖”区域。

### 8.4 拾遗 Historical Notes

目标：把画作背景、版本流传和图像考证组织成可信、可追溯、可持续更新的知识层。

核心需求：

- 内容分为“画作概览”“图像线索”“流传考证”“参考来源”“未定问题”。
- 所有关键事实必须有来源字段；无来源内容标记为“待考证”或“推测”。
- 不把学术内容写成百科堆砌，优先围绕用户在游湖中看到的节点展开。
- 支持从节点卡片跳转到对应拾遗条目，也支持从拾遗条目回到画卷位置。
- 使用证据分级：已证实、较可信、待考证、创作性解读。
- 可与 `问湖` 联动：拾遗中未解决的问题可生成问湖讨论入口，问湖中被验证的线索可沉淀回拾遗。

MVP 验收：

- 至少建立拾遗版块的信息结构和 3-5 条示例条目框架。
- 示例条目可以是“待补史料”的结构化占位，但不得伪造引用。
- 每条条目都能说明它关联哪个画卷节点或地图位置。

### 8.5 问湖 Cold Knowledge / Inquiry

当前初稿目标：把 `问湖` 先落成“西湖繁胜冷知识集”图册墙，让用户以轻阅读方式浏览南宋西湖风物冷知识，并通过整包下载和大图预览保存、细看卡片内容。

当前实现需求：

- 分区导航：标题区保留 `湖山岁时`、`湖上烟火`、`卷中微识` 三个纸签按钮，点击后平滑滚动到对应区块。
- 首屏密度：每个区块默认最多显示前 8 张卡片；有剩余真实卡片的区块显示“更多 N 张”，展开后追加剩余卡片并切换为“收起”。
- 真实资源：`湖山岁时` 为 24 张 PNG，`湖上烟火` 为 12 张 PNG，`卷中微识` 暂为 4 张 PNG；已配置整包的区块保留整包下载入口。
- `卷中微识`：当前 4 张正式卡片分别对应断桥、大佛头、茶司、湖上外卖；其他占位卡片已删除。
- 大图预览：点击真实知识卡片后打开跨区块预览轮播，覆盖 40 张真实图片，提供上一张/下一张、滚轮切换、标题和计数。
- 下载口径：卡片网格不显示单张下载按钮；单张保存交给浏览器右键，整包下载通过区块级入口完成。

长期目标：把 `问湖` 从冷知识展示继续扩展为面向公众和研究者的互动共研版块，让用户可以提出关于南宋西湖的问题、讨论知识点、提交线索或资料，并帮助项目持续补全考证层。

建议定位：`问湖` 不是普通评论区，而是“问题池 + 线索库 + 讨论区 + 共创任务”的轻社区。这样比单纯留言更适合本项目的学术和公众共创气质。

核心需求：

- 留下疑问：用户可提交关于画中地点、人物、建筑、舟船、题跋、时代背景、南宋西湖空间的问题。
- 讨论知识点：用户可围绕某个节点、拾遗条目或地图点展开讨论。
- 提供线索或资料：用户可提交文献线索、图像资料、地名线索、地方志摘录、论文/书籍信息、博物馆/馆藏链接等。
- 线索结构化：每条线索至少包含标题、类型、关联对象、说明、来源、可信度自评、提交者备注。
- 证据状态：所有社区内容默认不直接进入正式史料层，需经过“待核验 → 已采纳/未采纳/存疑”状态。
- 关联回路：问湖内容可关联 `入画` 地图点、`游湖` 节点或 `拾遗` 条目。
- 克制表达：界面应像“雅集/问答札记/线索笺”，避免社交媒体流、点赞排行、热榜等强娱乐化组件。

推荐内容类型：

| 类型 | 用途 | 示例字段 |
| --- | --- | --- |
| 问题 | 提出待考证疑问 | 问题、关联节点、疑问类型、补充说明 |
| 讨论 | 围绕知识点交流 | 主题、观点、依据、回应 |
| 线索 | 提交可核验资料 | 来源、摘录、链接/书目信息、关联位置 |
| 勘误 | 指出错误或不严谨处 | 错误位置、修正建议、依据 |
| 共创任务 | 项目方发布待协助事项 | 任务目标、所需资料、进度状态 |

推荐线索字段：

```yaml
id: inquiry-001
type: question | discussion | clue | correction | task
title: 标题
relatedTo:
  mapPoint: 可选地图点 id
  scrollNode: 可选游湖节点 id
  noteEntry: 可选拾遗条目 id
content: 正文
source:
  title: 来源标题
  author: 作者/机构
  year: 年份
  urlOrLocation: 链接/馆藏/书目信息/页码
status: 待核验 | 已采纳 | 未采纳 | 存疑
submitterNote: 提交者备注
createdAt: 创建时间
```

MVP 验收：

- 用户能理解 `问湖` 当前是“西湖繁胜冷知识集”，并能区分 `湖山岁时`、`湖上烟火`、`卷中微识` 三个分区。
- `湖山岁时`、`湖上烟火` 首屏均显示 8 张真实卡片，可展开查看剩余卡片，也可收起回到首屏密度；`卷中微识` 暂显示 4 张真实卡片且不显示占位。
- 点击任意真实卡片可打开大图预览轮播，预览中能切换上一张/下一张，并显示当前图片在 40 张真实卡片中的计数。
- 已配置整包下载的真实图片区块入口可见；单张卡片不显示序号、标题或下载按钮。
- 社区提问、线索提交、审核后台和待核验状态不作为当前 P0 验收项，保留为后续共研扩展。

## 9. 视觉与体验原则

本项目视觉系统遵循 `中式美学设计规范.md`，但要根据长卷网站场景做适配。

### 9.1 核心调性

- 素雅克制：以宣纸、米纸、浅纸和墨色层次做底，不使用现代高饱和大色块。
- 古籍风骨：标题、短引、节点题签可使用竖排文字和疏朗字距，形成古籍刻本般的秩序。
- 朱印点睛：朱砂红只用于关键操作、当前状态、节点强调、地图可进入点和印章式标识。
- 留白游观：界面元素必须让位于画卷和地图，不堆叠卡片、装饰和无效信息。
- 共研雅集：`问湖` 的社区体验应像札记、笺纸和雅集问答，而不是现代社交媒体信息流。
- 动效入境：当前 Landing 以静态底图和入口 hover 为主；如后续补动效，必须服务已确认底图，不重新打散版式。

### 9.2 颜色约束

建议沿用以下 token：

- 主朱：`#b83b3b`，用于版块入口 hover、当前导航、印章、地图点当前态、热点当前态。
- 北宋蓝：`#4a6b8c`，可用于历史时期/考证标签中的冷色辅助。
- 南宋绿：`#5c7a6b`，可用于地图自然层或可信度标签辅助。
- 沙金：`#c4a77d`，只作细节高亮，不做大面积铺色。
- 米纸：`#f4f1e8`，常规背景。
- 浅纸：`#faf9f5`，信息层背景。
- 纸边框：`#d4c4b7`，替代现代阴影。
- 焦墨：`#2c2c2c`，标题和正文主文字。
- 浓墨：`#6b6458`，次级说明。
- 淡墨：`#a89e89`，英文标签、辅助线和弱提示。

### 9.3 字体与排版

- 中文主字体优先使用 `Noto Serif SC` 或同气质宋体/衬线体。
- 英文、编号、坐标、进度等小标使用等宽字体，如 `JetBrains Mono`。
- 中文标题字距保持疏朗，重要题签可使用竖排。
- 节点卡片正文要可读，不为了竖排牺牲阅读效率；短题签竖排，长正文横排。
- 社区讨论正文以横排为主，问题题签、分类、状态可用竖排或印章式标签。

### 9.4 组件规范

- 信息卡片采用纸张边框，不用现代投影。
- 圆角控制在 0-2px，避免大圆角卡片感。
- 弹窗遮罩可使用温润深褐遮罩，但不能重到破坏画面细节。
- 按钮以文字、细边框、朱砂当前态为主，少用图标；必要图标优先极简线条。
- 地图点、社区状态、节点热点使用同一套朱印/淡墨标记语言，形成系统一致性。
- 禁用 emoji、玻璃拟态、蓝紫渐变、过度光效、赛博风和模板化 AI 国风装饰。

## 10. 交互设计要点

### 10.1 Landing 入境

- 首次访问默认展示 Landing Page，用户点击四个版块入口进入对应页面，默认主路径仍建议从 `入画` 开始。
- Landing 当前不设置主 CTA；四个入口必须始终可发现。
- 当前不提供“跳过动效/直接进入”能力；如后续新增较长动效，再补充跳过策略。
- Landing 的版块导览可作为次级入口，允许用户直接进入 `游湖`、`拾遗` 或 `问湖`，但默认路径仍是 `入画`。
- 低性能桌面环境允许降级为静态封面 + 轻量入场，不影响核心访问。

### 10.2 入画地图进入

- 地图点 hover/focus：显示点名、区域、进入提示和考证状态。
- 地图点 click/tap：进入 `游湖` 并跳转到对应区域、`scrollX` 或最近节点锚点。
- 未开放区域：以淡墨、低透明度和“待展开”状态表达，不做强按钮。
- 地图缩放：MVP 可先不做自由缩放，优先保证点击路径清楚。
- 返回路径：`游湖` 中应能返回 `入画` 地图，并保留当前区域状态。

### 10.3 长卷移动

- 桌面：鼠标滚轮默认映射为横向移动；Shift + 滚轮保持浏览器习惯兼容。
- 拖拽：按住画面可拖动画卷，鼠标指针状态需明确。
- 触控：移动端横向滑动和手势冲突处理后置，不纳入当前 MVP。
- 键盘：左右键移动，Home/End 回到段首/段尾，Esc 关闭卡片。

### 10.4 节点卡片

信息卡片字段建议：

```yaml
id: north-bank-001
title: 节点名称
subtitle: 图像观察或位置提示
scrollX: 横向位置百分比或像素坐标
mapPoint: 关联地图点
category: 建筑/桥梁/舟楫/人物/题跋/山水/待分类
summary: 80-150 字说明
sourceStatus: 已证实/较可信/待考证/创作性解读
sources: []
audioZone: 可选声景区域
relatedQuestions: 可选问湖问题 id 列表
```

### 10.5 声景

- 当前初稿不展示声景按钮、声景图标或声景状态。
- 声景作为后续增强项保留，不再属于当前 P0。
- 如后续恢复，必须先确认入口位置、音频授权、默认关闭策略和关闭方式。
- 声景不应喧宾夺主，以低音量环境声为主。
- 音频随滚动位置切换或交叉淡入淡出，而不是简单循环一首音乐。
- 所有声音都需要可关闭；未来可提供字幕/文字说明以照顾无声浏览场景。

后续可选声层：

- 基础层：水声、微风、远处人声。
- 区域层：舟楫、水岸、市井、寺钟、鸟鸣等。
- 交互层：打开节点时的极轻纸张/印章反馈音，必须克制。

### 10.6 问湖共研

- 从游湖节点或拾遗条目进入问湖时，应自动带上关联对象。
- 提交线索时，优先结构化来源字段，避免只留下模糊描述。
- 社区内容默认进入“待核验”，经人工确认后再进入拾遗或节点正式内容。
- 后续共研版本可采用“静态展示 + 表单占位 + 数据结构预留”起步；真实提交和审核后台可继续后置。

### 11.1 素材目录

- 原始素材：`data/raw/西湖北岸/`
- 入画鸟瞰标注图：`data/raw/西湖北岸/标注版本西湖全景鸟瞰.jpg`

- 处理后素材：`data/processed/西湖北岸/`
- Web 可用素材：后续根据技术方案生成，建议放入应用内 `public/assets/` 或等价目录。
- 导出图像：`exports/images/`

### 11.2 图片处理原则

- 原始大图不直接用于生产页面。
- 当前已生成 WebP 展示派生图，保证 Landing 首屏、入画地图、横卷和知识卡片优先走轻量展示素材；后续仍建议生成 AVIF 或分块瓦片图，进一步优化横向漫游性能。
- `入画` 当前采用已确认参考图作为底图，不再重新生成独立数据层地图；热区使用透明按钮覆盖红色分卷文字区域。
- `标注版本西湖全景鸟瞰.jpg` 和当前 `design/` 参考图用于确认鸟瞰入口点与跳转关系；后续工程化时可再把热区数据外置。
- `南宋西湖北线加标注*.jpg` 用于后续精修游湖节点位置和标注文本；当前初稿以三段北岸横卷可见文字热区和人工校准题签为准。
- 保留原图，不覆盖原始素材。
- `西湖北岸mini.jpg` 当前文件体积与大图接近，命名和用途需要复核。

### 11.3 史料内容原则

- 事实、推测、审美解读必须分层。
- 所有来源要记录标题、作者/机构、出处、年份、链接或馆藏信息、摘录位置。
- 没有来源时，不写“据记载”等模糊表达。
- 节点卡片优先短说明，深入内容放入拾遗版块。
- 问湖线索必须经过核验，才能进入正式拾遗或节点说明。

### 11.4 社区内容原则

- 问题与线索要可关联：至少关联地图点、游湖节点、拾遗条目或“全局问题”。
- 用户贡献内容不默认代表项目结论。
- 需要保留贡献状态：待核验、已采纳、未采纳、存疑。
- 后续若开放真实用户提交，需要增加隐私、版权、审核和反垃圾策略。

## 12. 成功指标

### 12.1 MVP 定性指标

- 用户能感受到“先题名入境，再俯瞰南宋西湖，最后进入长卷游湖”的空间转换。
- 用户能理解西湖北岸只是 MVP 范围，并期待扩展到全卷。
- 用户能从节点卡片获得有用信息，但不觉得被弹窗打断。
- 用户能理解 `问湖` 当前是冷知识图册入口，而不是普通评论区；后续共研能力作为扩展预期存在。
- 中式视觉气质克制、雅正、纸墨感明确，没有赛博国风或模板化 AI 感。

### 12.2 MVP 定量指标

首版可先用本地/测试环境人工验收，后续接入埋点：

- Landing 首屏可交互时间：目标 3 秒内，受素材处理和动效复杂度影响可阶段性调整。
- Landing 进入成功率：测试用户 90% 以上能找到四个入口，并能进入 `入画`。
- 入画地图进入成功率：测试用户 90% 以上能从鸟瞰入口点进入游湖对应区域。
- 当前初稿节点定位准确率：三段北岸横卷中的 15 个节点题签可定位、可点击并打开信息卡片；后续以“题签覆盖底图同名文字”为精修指标。
- 游湖区滚动帧率：桌面目标接近 60fps，低性能设备不低于可接受流畅度。
- 节点卡片打开成功率：测试用户 90% 以上能自行打开并关闭。
- 声景当前不纳入初稿指标；如恢复，再补充开启/关闭可发现性指标。
- 问湖图册理解率：测试用户能区分 `湖山岁时`、`湖上烟火`、`卷中微识` 三个分区，并能理解 `卷中微识` 当前为 4 张首批正式卡片，且能完成“更多展开”和大图轮播预览。

## 14. 风险与对策

| 风险 | 影响 | 对策 |
| --- | --- | --- |
| 原始图片体积过大 | 首屏慢、桌面卡顿、后续移动端不可用 | 生成 Web 分辨率图和瓦片，不直接加载原图 |
| 标注参考图被误用为生产图 | 页面出现烘焙文字、无法响应式控制热点 | 标注图只用于录入坐标和文本，生产层使用无标注素材 + 数据层热点 |
| 入画入口点与游湖细节点混为一谈 | 鸟瞰图过密、用户误以为所有细节点都应在地图上可见 | 入画只保留鸟瞰图重要入口点，游湖再展示 15 个细节点 |
| Landing 动效过重 | 用户进入主体验前等待过久或桌面首屏卡顿 | 当前保持静态底图和轻量 hover；如新增动效，四个入口优先可交互 |
| 移动端当前展示问题较多 | 若强行纳入 MVP 会拖慢桌面 Web 交付并增加返工 | MVP 明确桌面 Web-only；移动端作为后续专项重新设计与验收 |
| 入画地图被误解为精准复原 | 用户误解南宋西湖空间 | 标注“示意性”，用证据状态区分史实、推测和设计推演 |
| 史实来源不足 | 内容可信度下降 | 用“待考证/推测/已证实”分级，不伪造来源 |
| 横向滚动不符合用户习惯 | 用户迷失或误操作 | 提供进度、地图返回、键盘和拖拽提示，但提示要克制 |
| 声景入口未经确认就恢复 | 干扰当前已确认的纯画卷游览体验 | 声景不作为当前 P0；如恢复需先确认入口位置、授权音频和关闭策略 |
| 中式美学变成装饰堆砌 | 视觉俗套、AI 感强 | 严守纸墨、留白、朱印点睛和小圆角规范 |
| 问湖社区内容质量不稳定 | 噪音、误导或伪史料进入正式内容 | 默认待核验，人工采纳，正式内容与社区内容分层 |
| 用户提交资料涉及版权/隐私 | 后续发布风险 | 首版先做结构和样例，真实提交前补版权与审核规则 |

## 17. 当前假设

- 首版 MVP 以桌面 Web 体验为唯一验收设备范围；移动端可浏览不再作为当前承诺，后续通过响应式专项补齐。
- Landing Page 是进入四个一级版块前的独立入口，默认主路径为 Landing → 入画 → 游湖。
- Landing 已改为直接使用当前确认底图；`landing-page-web-effect-v2` 只作为历史参考，不再约束当前底图版式。
- `入画` 地图是示意性南宋西湖鸟瞰图，不是精准 GIS 复原。
- `入画` 地图点点击后进入 `游湖` 对应区域、`scrollX` 或最近节点锚点。
- `入画` 地图点不需要覆盖全部 15 个游湖节点；15 个 MVP 节点主要在 `游湖` 长卷中展示。
- 游湖区当前采用有限横卷浏览，不实现无缝循环。
- 当前初稿保留确认底图和 `巨石山下大石佛.jpg` 等源图，但运行时展示优先使用 WebP 派生图；正式长期发布前再推进瓦片化。
- 中式美学规范是本项目默认视觉约束，后续如有更具体品牌规范，以项目内更新文档为准。
- 史料内容必须先证据化，再写入正式说明。
- 15 个 MVP 节点名称已作为游湖题签与卡片清单落地，后续继续校准坐标和补正式史料来源。
- 声景正式素材和入口未定前，不在当前初稿中展示声景按钮。
- `问湖` 首版优先做冷知识图册与静态资源展示；真实社区后台、提交审核和线索沉淀可后置。
