## 1. gstack 设计审计结论

### 1.1 总体判断

当前 demo 的视觉方向成立：题名入境、纸墨质感、朱砂点睛、长卷游观和冷知识图册都有明确记忆点。它不像普通文旅官网，也避开了常见 AI 式 SaaS 卡片堆叠。

工程化前的主要问题不是“风格不够”，而是“规范不够可执行”：旧 `design.md` 更像视觉备忘录，缺少状态表、组件合同、桌面布局规则、素材加载失败口径、无障碍交互和工程验收标准。工程化网站必须把这些变成可以实现、测试、复盘的设计系统。移动端已确认后置，本版只定义桌面 Web MVP 的发布标准，并保留移动端问题作为后续专项输入。

## 2. 产品体验分类

本项目是 HYBRID：

- Landing 是品牌/沉浸入口，按海报式 Landing 规则处理。
- `入画` 是空间导航页，按交互地图和导览规则处理。
- `游湖` 是核心工具型画卷浏览器，按 App UI 规则处理，画卷是工作区。
- `拾遗` 是研究阅读页，按内容阅读和证据系统处理。
- `问湖` 当前是图册墙，按高密度视觉库处理；后续共研版本再按表单和线索库处理。

通用原则：

- 第一屏必须让用户知道“这是《西湖繁胜全景图》”。
- 每个版块只有一个主要任务。
- 卡片只有在“卡片就是内容对象或交互对象”时出现。
- 不用现代 SaaS 三列功能卡、胶囊按钮、蓝紫渐变、玻璃拟态、emoji 装饰和大圆角。

### 3.1 页面层级

```text

| Landing：题名入境                                         |
|  - 第一视觉：西湖繁胜全景图                               |
|  - 主操作：入画 / 游湖 / 拾遗 / 问湖                       |
+------------------------------------------------------------+
              |
              v
+------------------------------------------------------------+
| App Shell：统一抬头                                       |
| 入画：全湖导览 -> 游湖分卷入口                             |
| 游湖：横卷浏览 -> 节点题签 -> 节点卡片                     |
| 拾遗：画作背景、证据、流转、研究入口                       |
| 问湖：冷知识图册 -> 更多展开 -> 大图预览                   |
+------------------------------------------------------------+
```

### 3.2 首屏三件事

| 页面 | 用户必须先看到 | 第二优先级 | 第三优先级 |
| --- | --- | --- | --- |
| Landing | 项目题名和南宋西湖气质 | 四个版块入口 | 底部长卷氛围 |
| 入画 | 南宋西湖环湖导览图 | 可进入的分卷文字 | 当前导航态 |
| 游湖 | 横卷图像本体 | 朱砂节点题签 | 底部位置滑块 |
| 拾遗 | 画作/项目背景 | 证据与时间线 | 关联画卷局部 |
| 问湖 | 西湖繁胜冷知识集标题 | 三个分区纸签 | 首屏知识卡片 |

### 3.3 用户旅程

| 步骤 | 用户动作 | 用户感受目标 | 设计支撑 |
| --- | --- | --- | --- |
| 1 | 打开网站 | 停顿、识别、被邀请入卷 | Landing 海报式构图、题名最大化、入口克制 |
| 2 | 进入入画 | 建立方位感 | 全湖示意图、分卷文字、透明热区 |
| 3 | 进入游湖 | 像展开长卷一样观看 | 横卷铺满、UI 后退、滑块和拖拽 |
| 4 | 点击节点 | 细节被解释但不打断 | 朱砂题签、纸笺弹层、Esc/遮罩关闭 |
| 5 | 查看拾遗 | 从观看进入考证 | 札记版式、来源层级、证据状态 |
| 6 | 浏览问湖 | 轻阅读、收藏、继续发现 | 8 列图册、更多展开、大图轮播、整包下载 |

时间尺度：

- 5 秒：用户知道这是西湖长卷项目，并找到入口。
- 5 分钟：用户完成入画、游湖、节点卡片和问湖预览的闭环。
- 5 年：文档、数据和组件足够稳定，能继续扩展全卷、史料和共研内容。

## 4. 设计原则

1. 画卷优先：`游湖` 中所有 UI 必须让位于横卷本体。
2. 纸墨为底：页面以米纸、浅纸、焦墨、淡墨和旧纸纹理构成，不使用现代高饱和底。
3. 朱砂点睛：朱砂只用于当前态、入口、节点、印章、关键操作和状态提示。
4. 竖排有边界：短题签、入口、章序可竖排；正文、说明、表单、来源必须横排。
5. 一屏一事：每个页面首屏只承担一个核心任务。
6. 数据可替换：背景图、节点、图册、来源、状态全部应能外置为数据。
7. 状态可见：加载中、空内容、失败、部分成功、完成都要有明确视觉反馈。
8. 无障碍可测：键盘、读屏、对比度和 reduced motion 都是桌面 MVP 验收项；触控体验后置。

## 6. 桌面 Web 视口系统

| 断点 | 宽度 | 设计策略 |
| --- | --- | --- |
| mobile/post-MVP | 320-599 | 后续专项；当前只记录问题，不进入 MVP 验收 |
| tablet/post-MVP | 600-1023 | 后续专项；可作为响应式阶段中间断点 |
| desktop | 1024-1439 | 标准体验，图册 4-8 列按容器调整 |
| wide | 1440+ | 保持内容最大宽度或画卷铺满，不把正文拉太宽 |

MVP 强制规则：

- 当前验收只覆盖 `1024px` 及以上桌面 Web 视口。
- `1440x900` 是主视觉校验视口，`1024x768` 是最小桌面可用视口。
- App Shell 在桌面端保持单行导航，不遮挡主内容。
- `游湖` 桌面端必须支持鼠标拖拽、滚轮缩放、底部滑块和键盘左右键。
- `问湖` 桌面端优先 8 列图册；低于 1200px 可收缩为 4-6 列，但不进入移动 2 列模式。

后续移动端专项再处理：

- 触控目标 `44x44px`。
- App Shell 两行或底部导航。
- Landing 独立入口组。
- `问湖` 2 列图册与移动预览控件。
- `游湖` 横向滑动、节点列表辅助和浏览器返回手势冲突。

## 7. 组件规范

### 7.1 AppShell

职责：统一品牌识别、一级导航和当前态。

结构：

- 左侧：朱印 wordmark + 项目名 + `Digital Scroll`。
- 右侧：`入画`、`游湖`、`拾遗`、`问湖`。
- 最右：圆形山水章或全屏/视图状态入口，当前仅装饰时不可误导为按钮。

状态：

| 状态 | 视觉 |
| --- | --- |
| default | 纸底、细线、浓墨导航 |
| current | 朱砂小线或点，不铺色块 |
| hover/focus | 朱砂文字 + 明确 focus outline |
| reduced-motion | 不播放抬头动效 |

工程要求：

- 当前 MVP 保持原 demo 抬头实现，不在本轮用 DOM + token 重写。
- 透明可聚焦热区和可读 aria label 是在不改变原 demo 视觉的前提下补齐的工程要求。
- DOM + token 抬头只能作为未来专项，启动前必须完成与原 demo 的逐屏对照和用户确认。
- 当前态不得只靠背景图表达，必须有 DOM 状态类。

### 7.2 LandingEntry

职责：从卷首进入四个版块。

规则：

- 桌面端入口可覆盖在确认底图上，但命中区域应大于可见文字。
- hover/focus 只做朱砂变色或轻微强调，不重排底图。
- 移动端入口组后置到响应式专项，不作为当前 MVP 实现要求。

### 7.3 MapHotspot

职责：在 `入画` 鸟瞰图上表达可进入分卷。

规则：

- 透明热区覆盖底图红色分卷文字区域。
- hover/focus 可显示极淡朱砂框、cursor 和 title，不显示大浮层。
- 待开放热区不能跳到空白游湖页；应显示待开放状态或禁用。
- 热区数据必须外置为 `map-points` 结构，包含 `id`、`label`、`status`、`targetType`、`scrollTarget`、`position`。

### 7.4 ScrollViewport

职责：横卷浏览的主工作区。

规则：

- 横卷图像本体是主角，铺满抬头下方空间。
- 桌面：拖拽平移，滚轮缩放，底部 range 控制横向位置。
- 移动：后续专项再定义，不纳入当前 MVP。
- 未加载完成时不得出现空白米纸区和悬空题签，必须显示加载状态。

### 7.5 ScrollMarker

职责：替代画中题字的朱砂纸签。

规则：

- 文字逐字单列，不自动分两列。
- 优先遮住底图同名文字，不遮挡建筑主体和人物。
- 题签可随 zoom 缩放，桌面命中区域必须稳定；移动端 44px 触控目标后置。
- 可被键盘聚焦，focus 状态明显。

### 7.6 NodePanel

职责：解释游湖节点。

结构：

- 标题：横排大字，长标题缩小或去掉括号别名。
- 状态签：如“南宋 · 西湖堤桥”。
- 拼音小标。
- 三段位置标签。
- 四段正文：【节点简介】【南宋建制】【今日坐标】【史迹价值】。
- 右下角朱印。

交互：

- 打开：点击题签、Enter、Space。
- 关闭：关闭按钮、Esc、点击遮罩。
- 焦点：打开后焦点进入关闭按钮，关闭后回到触发题签。
- 背景：遮罩温润，不把画卷压成纯黑。

### 7.7 NotesSurface

职责：承载画作背景、证据、流转与研究说明。

当前原型为整页参考图。工程化版本应分两层：

- 展示层：保留当前整页参考图作为视觉方向。
- 数据层：后续把简介、时间线、来源和证据状态拆为结构化内容。

要求：

- 正文必须可复制、可搜索、可读屏。
- 图片化文字只能作为视觉参考，不作为唯一内容来源。
- 来源、证据状态和关联节点必须可追踪。

### 7.8 ColdKnowledgeGallery

职责：`问湖` 冷知识图册。

结构：

- Hero：`WEST LAKE MISCELLANY`、`西湖繁胜冷知识集`、一句说明。
- 分区纸签：`湖山岁时`、`湖上烟火`、`卷中微识`。
- 区块头：序号、标题、副标题、说明、操作区。
- 网格：真实图片卡片；后续缺图时才使用明确“待补”的占位卡片。

规则：

- 桌面默认 8 列；移动 2 列后置到响应式专项。
- 每区块首屏最多显示 8 张，不足 8 张时按实际卡片数量显示。
- 有真实剩余卡片时显示“更多 N 张”，展开后变为“收起”。
- `卷中微识` 当前显示 4 张正式知识卡片，不再显示占位。
- 图片卡片不显示序号、标题或单张下载按钮。

### 7.9 ColdPreviewCarousel

职责：查看 40 张真实冷知识卡片大图。

规则：

- 遮罩层覆盖页面，图片居中完整显示。
- 两侧上一张/下一张按钮固定在预览层，不进入图册网格。
- 底部显示 `分区 · 卡片标题` 和 `当前 / 总数`。
- 支持点击背景、关闭按钮、Esc 关闭。
- 支持左右键、按钮、滚轮切换。
- 桌面端优先完整显示图片与计数；移动端预览布局后置。

## 9. 交互规范

### 9.1 路由

| 路由 | 目标 |
| --- | --- |
| `#landing` | 卷首入口 |
| `#map` | 入画导览 |
| `#scroll` | 游湖默认位置 |
| `#scroll?target={nodeId}` | 游湖指定节点或分卷 |
| `#notes` | 拾遗 |
| `#inquiry` | 问湖 |

路由要求：

- hash 切换后更新 `body[data-view]` 和当前导航态。
- 深链进入 `#scroll?target=...` 时，图像加载完成前应先显示定位中状态。
- 无效 target 回到安全默认位置，并显示轻提示。

### 9.2 键盘

| 区域 | 键盘行为 |
| --- | --- |
| AppShell | Tab 聚焦，Enter 进入版块 |
| MapHotspot | Tab 聚焦，Enter/Space 进入目标 |
| ScrollViewport | 左右键移动，Home/End 到首尾，Esc 关闭卡片 |
| ScrollMarker | Enter/Space 打开节点卡片 |
| NodePanel | Esc/关闭按钮关闭，焦点循环在弹层内 |
| ColdGallery | Tab 到卡片，Enter/Space 打开预览 |
| ColdPreview | 左右键切换，Esc 关闭，焦点在弹层内 |

### 9.3 指针输入

- 拖拽画卷时 cursor 从 grab 变 grabbing。
- 滑块变化、拖拽和键盘移动都必须更新同一个归一化位置。
- hover 不是唯一反馈，focus 必须等价。
- 触控手势、移动端 hover 替代和移动端返回手势冲突后置。

### 9.4 动效

| 动效 | 时长 | 规则 |
| --- | ---: | --- |
| 页面切换 | 160-240ms | 淡入或轻位移，不旋转、不爆炸 |
| 节点卡片 | 180-240ms | 纸笺淡入或边缘轻推 |
| 题签 hover | 120-180ms | 朱砂增强，不跳动 |
| 图册卡片 hover | 140-180ms | 上浮不超过 4px |
| 预览切换 | 120-200ms | 快速淡入，不影响阅读 |

`prefers-reduced-motion: reduce` 时：

- 保留 opacity 变化。
- 禁止视差、长距离平移和自动播放。
- 所有操作立即完成。

## 10. 无障碍规范

### 10.1 语义

- 页面使用 `header`、`main`、`section`、`nav`。
- 只有一个当前页面主标题可被读屏识别。
- 图片必须有准确 `alt`，装饰图 `aria-hidden="true"`。
- 节点卡片和大图预览使用 `role="dialog"`、`aria-modal="true"`、`aria-labelledby`。

### 10.2 可操作性

- 所有按钮、热区、题签、卡片、关闭控件可键盘访问。
- 禁止只用透明按钮且无焦点样式。
- 焦点可见，不被固定抬头遮挡。
- 弹层打开时焦点不泄漏到背景页面。

### 10.3 可感知

- 文本不小于 16px，辅助英文不承担核心信息。
- 图片化正文必须有 DOM 文本备份。
- 颜色不作为唯一状态提示。
- 大图预览计数必须可读，不叠在复杂图像上。

## 11. 素材与内容规范

### 11.1 资产分层

| 类型 | 目录建议 | Git 策略 | 用途 |
| --- | --- | --- | --- |
| 原始素材 | `data/raw/` | 默认不提交，除非确认 | 原始图、PSD、未压缩文件 |
| 处理素材 | `data/processed/` | 默认不提交 | 切片、测算、中间图 |
| Web 素材 | `app/assets/` 或后续 `public/assets/` | 可提交 | 前端加载 |
| 导出物 | `exports/` | 默认不提交 | zip、pdf、图片包 |

工程化要求：

- 页面不得直接依赖 100MB 级原始图作为生产资源。
- Landing、入画、游湖、问湖卡片都要有 Web 优化版本。
- 每个图片资源需记录宽高、格式、来源、压缩参数和是否可发布。
- 长卷建议使用多分辨率切片或瓦片加载。

### 11.2 缺图与慢加载

- 缺图不是空白，必须显示纸底占位、资源名称和可恢复操作。
- 慢加载时保留稳定尺寸，避免布局跳动。
- 图册卡片使用固定 `aspect-ratio: 1086 / 1448`。
- 横卷使用预设 `width`、`height`，加载完再替换高清图。

### 11.3 文案口径

- 不写无来源的确定史实。
- “待考证”“创作性推演”“较可信”“已证实”必须视觉区分。
- 页面内文案面向用户，不解释设计意图。
- 文档内可以解释设计规范，但应用界面不显示“如何使用”的大段说明。

## 12. 页面级验收

### 12.1 Landing

- 题名是第一视觉信号。
- 四个入口 3 秒内可发现、可点击、可键盘聚焦。
- 桌面入口命中区域稳定，不依赖用户精确点击极小文字。
- 背景失败时仍有可用入口。

### 12.2 入画

- 用户能理解这是南宋西湖环湖导览。
- 开放热区可进入有效游湖位置。
- 待开放热区不跳转到空白页。
- 桌面热区可鼠标点击、键盘聚焦，并有清楚 focus 状态。

### 12.3 游湖

- 横卷图像成功显影前有加载态。
- 横卷、题签、滑块位置一致。
- 题签不遮挡画作主体。
- 节点卡片可打开、关闭、键盘操作。
- 画卷加载失败时有可读错误和重试/返回路径。

### 12.4 拾遗

- 背景参考图之外有可读文本层。
- 来源和证据状态清楚。
- 内容可复制、可搜索、可读屏。
- 与游湖节点有明确关联。

### 12.5 问湖

- 三个分区入口清楚。
- 桌面图册一行最多 8 张，窄桌面可收缩为 4-6 列。
- 更多/收起不会跳丢用户上下文。
- 真实卡片可进入 40 张范围的大图预览。
- `卷中微识` 当前显示 4 张真实卡片，后续缺图再回到明确待补占位规则。

## 13. 工程化设计决策

### 13.1 已定

- 当前视觉方向保留：纸墨、长卷、朱砂、宋式留白。
- 声景不属于当前 P0。
- `问湖` 当前首版是冷知识图册，不是社区表单。
- `游湖` 是核心体验，工程化优先级高于装饰性页面。
- 卡片不作为通用布局容器，只作为内容对象、弹层或图册对象。

### 13.2 待工程确认

| 决策 | 推荐 | 若推迟的风险 |
| --- | --- | --- |
